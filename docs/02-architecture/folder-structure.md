## Purpose of Each Folder

### 00-governance
Defines how the project is managed and the principles that guide decisions.

| File                | Description                                                                                      |
| ------------------- | ------------------------------------------------------------------------------------------------ |
| **charter.md**      | Defines the project's purpose, goals, scope, and success criteria.                               |
| **glossary.md**     | Defines project-specific terminology, abbreviations, and naming conventions.                     |
| **orchestrator.md** | Explains the overall workflow for developing, documenting, testing, and maintaining the project. |
| **principles.md**   | Lists the core engineering and design principles that guide every decision.                      |

### 01-planning
Tracks what needs to be built, current priorities, and long-term goals.

| File                | Description                                                          |
| ------------------- | -------------------------------------------------------------------- |
| **backlog.md**      | Stores all future ideas, features, improvements, and technical debt. |
| **milestones.md**   | Tracks major project milestones and completion progress.             |
| **roadmap.md**      | Outlines the long-term development plan and project phases.          |
| **session-plan.md** | Defines the goals and priorities for the current work session.       |

### 02-architecture
Documents technical decisions and the overall organization of the project.

| File                    | Description                                                                   |
| ----------------------- | ----------------------------------------------------------------------------- |
| **decisions.md**        | Records important technical decisions and the reasoning behind them.          |
| **folder-structure.md** | Documents the project's directory structure and the purpose of every folder.  |
| **site-structure.md**   | Maps the website's pages, navigation, and overall information architecture.   |
| **technology.md**       | Lists the technologies, frameworks, libraries, and tools used in the project. |

### 03-checklists
Reusable procedures followed at the beginning and end of every work session, deployment, or release.

| File                 | Description                                                         |
| -------------------- | ------------------------------------------------------------------- |
| **session-start.md** | Checklist to prepare the project before beginning work.             |
| **session-end.md**   | Checklist to review, document, and safely finish each work session. |
| **deployment.md**    | Step-by-step checklist for deploying the website to production.     |
| **release.md**       | Final verification checklist before publishing a new version.       |

### 04-progress
Historical record of development, completed work, bugs, and lessons learned.

| File                   | Description                                                                 |
| ---------------------- | --------------------------------------------------------------------------- |
| **daily-log.md**       | Records what was accomplished during each development session.              |
| **change-log.md**      | Summarizes notable changes made throughout the project's history.           |
| **bugs.md**            | Tracks known bugs, their status, and planned fixes.                         |
| **lessons-learned.md** | Captures important insights and improvements discovered during development. |

### 05-learning
Personal notes and study material collected while working on the project.

| File                   | Description                                                                                               |
| ---------------------- | --------------------------------------------------------------------------------------------------------- |
| **css.md**             | Notes on CSS syntax, layout systems, styling techniques, animations, and best practices.                  |
| **git.md**             | Explains Git concepts, version control workflows, branching, merging, and common commands.                |
| **github.md**          | Documents GitHub repositories, workflows, collaboration, authentication, and project management features. |
| **github-pages.md**    | Explains how GitHub Pages works, deployment steps, custom domains, DNS, and HTTPS configuration.          |
| **html.md**            | Notes on HTML structure, semantic elements, accessibility, and document organization.                     |
| **js.md**              | Covers JavaScript syntax, programming concepts, DOM manipulation, events, and modern features.            |
| **terminal.md**        | Documents Terminal usage, shell commands, navigation, package management, and workflow tips.              |
| **web-development.md** | High-level concepts, development philosophy, and how all web technologies work together.                  |

### 06-design
Documents the visual language of the project, including colors, components, inspiration, and design standards.

| File                 | Description                                                                                 |
| -------------------- | ------------------------------------------------------------------------------------------- |
| **color-palette.md** | Documents the project's approved colors, usage, and branding guidelines.                    |
| **components.md**    | Catalogs reusable UI components and their intended use.                                     |
| **design-system.md** | Defines the project's visual language, spacing, layouts, icons, and interface standards.    |
| **inspiration.md**   | Collects design references, ideas, and visual inspiration for the project.                  |
| **typography.md**    | Documents fonts, font hierarchy, sizing, spacing, readability, and text styling guidelines. |

### 07-reference
Quick-reference material used frequently during development, including commands, resources, shortcuts, and solutions to common problems.

| File                   | Description                                                                       |
| ---------------------- | --------------------------------------------------------------------------------- |
| **commands.md**        | Stores commonly used terminal, Git, and development commands for quick reference. |
| **resources.md**       | Lists useful websites, documentation, tutorials, and external tools.              |
| **shortcuts.md**       | Documents keyboard shortcuts, editor tips, and workflow accelerators.             |
| **troubleshooting.md** | Records recurring problems, their causes, and proven solutions.                   |

### 08-documentation

Provides reusable starting points to ensure documentation remains consistent across every project.

| File                           | Description                                                                 |
| ------------------------------ | --------------------------------------------------------------------------- |
| **checklist-template.md**      | Standard template for creating consistent project checklists.               |
| **daily-log-template.md**      | Standard template for recording work completed during a development session. |
| **decision-template.md**       | Template for documenting technical decisions and the reasoning behind them. |
| **governance-template.md**     | Template for creating governance documents such as charters and principles. |
| **learning-template.md**       | Template for documenting new concepts, notes, and reference material.       |
| **roadmap-template.md**        | Template for planning project phases, milestones, and long-term goals.      |
